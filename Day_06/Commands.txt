0) Once you push into some registry if it's not dockerhub, the image __MUST__ include the storage address like
        localhost:5050/yaroslavdomb/image:version
    This is because command docker push will take only the image name(and it should already has registry address) and then parse it inside. 

1) Before sending layer to registry Docker will check his build hash. 
    In case it's already exist in Docker the layer will not be pushed again.

2) Local registry is working on HTTP only, while Docker expect registry with HTTPS only.
    There is 2 way to overcome it:
    - register local HTTP as insecure but allowed
    - 

    To register on Mac/Win you need to add into Docker Desktop -> Settings -> Docker Engine 
        {
            "insecure-registries": ["localhost:5000"]
        }
    For Linux need to add the same into /etc/docker/daemon.json

3) Image with couple of tags will remain until the last tag will not be removed.

4) Max tag length = 128. Cnnot include @ and :



//**************__ANTI-PATTERNS__*****************

1) Use with "latest" - you don't know behavior of the image/container
    docker pull yaroslavdomb/myapp:latest

2) Overwrite the version - you can breack currently working image/container
    docker tag myapp:2.0 yaroslavdomb/myapp:1.0
    docker push yaroslavdomb/myapp:1.0 

3) Push having no version
    docker push yaroslavdomb/myapp

4) Push having secrets in ENV/ARG inside the dockerfile


//**************__BEST_PRACTIES__*****************
1) Use & push 2 versions: with version as number and version as "latest"
    It will push only ones the file, but then you can be sure even you missed 
    the version, the latest will point on proper file

Task-2 (local repository)
    Start local private registry ("my_registry") on port 5050. Use volume /my/registry/data (check hub.docker.com for the correct internal data path). Config daemon.json to work with HTTP on localhost:5050. Build image from alpine and tag it as yaroslavdomb/day06:t2. Push it into the registry with numbered tag. Test existance of it with command "curl http://localhost:5050/v2/_catalog" (for Win "curl.exe http://localhost:5050/v2/_catalog").

Solution-2
    1) Dockerfile = see t2.dockerfile
    2) According to your OS push inside the file {"insecure-registries": ["localhost:5050"]}
    3) Start registry
        docker run --name my_registry --rm -v /my/registry/data:/var/lib/registry -p 5050:5000 registry:3
    4) Create image:
        docker build --build-arg VER=3.23.3 -t yaroslavdomb/day06:t2 -f .\t2.dockerfile .
    5) Push image into local registry
        docker tag yaroslavdomb/day06:t2 localhost:5050/yaroslavdomb/day06:1.0
        docker push localhost:5050/yaroslavdomb/day06:1.0
    6) Check repo:
        curl.exe http://localhost:5050/v2/_catalog
        #got {"repositories":["yaroslavdomb/day06"]}
    7) To get tags:
        curl.exe http://localhost:5050/v2/yaroslavdomb/day06/tags/list
