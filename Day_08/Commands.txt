//********************__BASIC__******************************
Security options:
    1) Change user from root to non-root
    2) Use Rootless Docker (there are some restrictions and config that need to  be done)
    3) Use Capabilities

1) CHANGING USER
    #Section in Dockerfile related to the user
        RUN useradd --no-create-home --shell /bin/false --uid 1000 appuser
        RUN chown -R appuser:appuser /app
        USER appuser

    Where:
        --no-create-home = less specific user space (Desctop/MyDocuments & so on) = more secure 
        --shell /bin/false = grant that user will not be interactive
        second command = switch from root:root to appuser:appuser for all the sub-folders of /app

2) ROOTLESS
    If you left the container and reach the host you'll get the same privileges as docker creator
    There are couple of limitations for rootless mode inside the container:
        - Not allowed ports under 1024(by  default, but could be allowed)
        - there is no --privileged mode
        - works slower due to additional checks and layers

3) CAPABILITIES
    #remove all caps given by default
    docker run --cap-drop ALL myapp

    #first - remove all caps, than add only needed
    docker run --cap-drop ALL --cap-add NET_BIND_SERVICE myapp
    # NET_BIND_SERVICE — allow bind for ports < 1024

    # In docker-compose.yml:
    services:
    app:
      cap_drop:
        - ALL
      cap_add:
        - NET_BIND_SERVICE


//********************__EXAMPLES__******************************

    # Start with specific UID, 1000 = first non-system UID number
    docker run --user 1000 myapp

    # Start using UID:GID
    docker run --user 1000:1000 myapp

    # Start as user XXX if exist in /etc/passwd
    docker run --user appuser myapp

    # Start as ROOT for check during develoment
    docker run --user root -it myapp bash

    # Start as current HOST user
    docker run --user $(id -u):$(id -g) myapp

TASK-1 
    Run container with and without user declaration:
        docker run --rm alpine whoami
        docker run --rm alpine id
        docker run --rm --user 1000 alpine whoami
        docker run --rm --user 1000 alpine id
        docker run --rm --user 1000:1000 alpine whoami
        docker run --rm --user 1000:1000 alpine id
Solution-1
    whoami = goes to etc/passwd and look for the name 1000. Tht's why you got "unknown uid 1000"
    id = it's more low level app, so it does not care about names in etc/passwd.

    It's also explain the diff between --user 1000 (created user only) and --user 1000:1000 (created user+group)

TASK-4
    Capabilities
        If you run 
            docker run --rm --cap-drop ALL nginx:alpine
        you'll get error CHOWN Operation not permitted
        Then let's try to add all needed:
            docker run --rm --cap-drop ALL --cap-add CHOWN --cap-add SETUID --cap-add SETGID --cap-add NET_BIND_SERVICE nginx:alpine
        Test if this is working with 
            docker run --rm -p 8080:80 --cap-drop ALL --cap-add CHOWN --cap-add SETUID --cap-add SETGID --cap-add NET_BIND_SERVICE nginx:alpine
            curl.exe http://localhost:8080
